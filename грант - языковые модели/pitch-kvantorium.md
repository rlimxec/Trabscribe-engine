## Слайд 2 • Проблема

# Осетинский язык в цифровой среде: положение критическое

## Статистика
| Показатель | Значение |
|---|---|
| Число носителей | ~345 000 человек |
| Статус по ЮНЕСКО | **«Уязвимый»** (Vulnerable) |
| Снижение владения за 10 лет | **–24%** |
| Первоклассники, не владеющие осетинским | **~80%** |

## Цифровой разрыв
- Ни один коммерческий ASR-сервис (Speech-to-Text) **не поддерживает** осетинский язык
- Яндекс SpeechKit — нет осетинского
- Сбер SaluteSpeech — нет осетинского
- Google Cloud STT — нет осетинского
- Vosk (open-source) — нет осетинского
- OpenAI Whisper (базовая модель) — нет осетинского

## Последствия
- Голосовые интерфейсы недоступны для осетинского языка
- Невозможна автоматическая расшифровка осетинской речи (архивы, лекции, новости)
- Язык вытесняется из технологической среды — один из ключевых факторов умирания языка в XXI веке

[SPEAKER NOTES]
345 тысяч носителей — это не мало. Для сравнения, исландский язык имеет около 360 тысяч носителей — и он полностью поддерживается всеми ASR-сервисами. Ключевая разница: Исландия вложилась в языковые технологии.

Осетинский теряет по 2.4% носителей ежегодно. Снижение на 24% за 10 лет — это данные переписей 2010 и 2020 годов. Основная причина — язык не представлен в цифровой среде: дети не печатают на осетинском, не общаются голосом с устройствами, не потребляют контент.

80% первоклассников не владеют осетинским — это данные Министерства образования РСО-Алания. Язык перестаёт передаваться в семье.

Коммерческие сервисы Яндекса, Сбера и Google не имеют осетинского в роадмапах — рынок слишком мал для них. Это означает, что единственный путь — открытая модель силами сообщества. Что мы и предлагаем.
[/SPEAKER NOTES]

## Слайд 3 • Решение

# Whisper Fine-Tuning: технологический подход

## Что такое Whisper

Whisper — модель Speech-to-Text от OpenAI (сентябрь 2022):
- **Open-source** (MIT-лицензия) — можно модифицировать, дообучать, распространять
- Поддерживает **99 языков** в базовой версии (но не осетинский)
- Архитектура Encoder-Decoder Transformer (аналог GPT для аудио)
- Обучена на 680 000 часов аудио (supervised + weakly supervised)
- Доступна в размерах: tiny (39M), base (74M), small (244M), medium (769M), large-v3 (1.5B)

## Как мы планируем адаптировать Whisper под осетинский

**Метод:** Parameter-Efficient Fine-Tuning (PEFT) + Low-Rank Adaptation (LoRA)

```
Базовая модель Whisper (small или medium)
         │
         ▼
  ┌─────────────────────┐
  │      LoRA-адаптер    │  ← замораживаем 97.16% параметров,
  │  (ранг r=16, α=32)   │     обучаем только 2.84%
  └─────────────────────┘
         │
         ▼
  ┌─────────────────────┐
  │ Осетинский          │
  │ tokenizer +         │
  │ аудио-препроцессор  │
  └─────────────────────┘
         │
         ▼
  ┌─────────────────────┐
  │ Осетинский          │
  │ Language Model      │
  │ (n-gram +          │
  │  осетинский словарь)│
  └─────────────────────┘
         │
         ▼
   ASR-выход: осетинский текст
```

### Ключевые цифры
| Параметр | Значение |
|---|---|
| Базовая модель | Whisper small (244M параметров) |
| Обучаемых параметров (LoRA) | ~6.9M (2.84%) |
| Необходимо аудио для fine-tuning | от 30 часов (чистого) |
| Оценочный WER (Word Error Rate) | < 30% после первой итерации |
| Целевой WER | < 15% |
| Формат | GGML / GGUF — запуск на CPU и Metal GPU |

## Итог
Первая open-source ASR-модель для осетинского языка, способная работать **offline** на обычном ноутбуке.

[SPEAKER NOTES]
Ключевое техническое решение — LoRA-адаптеры. Мы не обучаем модель с нуля — это потребовало бы десятки тысяч часов осетинского аудио и миллионы долларов. Вместо этого мы берём готовую multilingual-модель Whisper, замораживаем 97% весов и дообучаем только небольшую адаптивную прослойку.

Это стандарт индустрии: именно так адаптируют LLM под новые языки. LoRA была разработана Microsoft в 2021 году и стала де-факто методом для таких задач.

Целевой WER (Word Error Rate) менее 15% — это уровень, при котором транскрибация становится практически полезной. Для сравнения: WER человеческого транскрайбера — 4-5%, коммерческие ASR для массовых языков дают 7-12%.

Модель будет экспортирована в формат GGUF — это значит, она запускается через llama.cpp / whisper.cpp на любом устройстве: от MacBook до Raspberry Pi, полностью offline, без отправки данных в облако.
[/SPEAKER NOTES]

## Риски и их mitigation

| Риск | Вероятность | Mitigation |
|---|---|---|
| Недостаточно осетинского аудио | Средняя | Есть 44h из датасетов + Synthetic Data Generation (TTS → ASR) |
| WER выше целевого (неудовлетворительное качество) | Средняя | Итеративный подход: дообучение на лучших данных, ensemble моделей |

| Смена наставника | Средняя | Документирование процессов, минимум 2 обученных человека |
| Отказ грантового фонда | Высокая | Диверсификация: подача в 3-4 фонда одновременно |

[SPEAKER NOTES]
Дорожная карта реалистична: 5 месяцев от старта до полевого теста.

Важно про риски. Основной — качество модели. Но это решается итеративно: первая версия с WER 30% уже полезна (можно понять общий смысл). Вторая версия — WER < 20%. Третья — WER < 15%.

Грантовый риск мы mitигируем диверсификацией: подача в 4-5 фондов одновременно. Даже если откажут 3 из 5 — проект всё равно идёт на личных средствах.

Школьники не теряют интерес, если видят реальный результат. Это не абстрактная задача — они будут записывать свой язык и видеть, как модель учится его понимать. Через месяц они показывают друзьям: Siri не понимает осетинский — а наша программа понимает. Это мощная мотивация.
